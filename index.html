▶ 내가 담당했던 부분

저는 팀 프로젝트에서 인증/인가 시스템을 구현하였습니다. 먼저, 토큰 생성 시 env를 활용하여 환경 설정을 가져와 특정 깃 주소에서 data.SECRET_KEY를 추출하여 토큰을 생성합니다. 이렇게 생성된 토큰은 Spring Cloud Gateway에서 기본적으로 제공하는 필터를 커스텀하여 활용합니다. 구현된 커스텀 필터는 인증이 필요한 특정 CRUD 작업을 수행할 때마다 실행됩니다. 클라이언트의 요청이 도착하면 해당 필터에서는 Authorization 헤더의 존재와 토큰의 유효성을 검사합니다. 검증에 실패하면 401에러로 응답하며, 검증에 성공하면 요청을 다음 처리 단계로 전달하여 작업을 계속할 수 있도록 합니다.

Kerly-PROJECT_App



▶ 마켓켈리
- 회원인증(auth), 즐겨찾기(bookmark), 파일관리(file), 상품관리(item), 사용자 관리(member), 주문관리(newOrder), 댓글관리(reply), 보안관리(sec) 패키지를 포함한 Monolithic Architecture(MA)백엔드

- 안드로이드 스튜디오를 통해 Flutter를 활용한 어플리케이션

마켓컬리 앱을 벤치마킹한 '켈리 프로젝트' 입니다. 이 프로젝트는 친환경 음식 소재 쇼핑몰입니다. 환경 보호와 건강한 라이프스타일을 촉진하며, 사람들에게 친환경적인 선택의 기회를 제공하는 것을 목표로 합니다.

▶ 내가 담당했던 부분

저는 이 프로젝트의 핵심인 배달 서비스를 구현했습니다. 사용자의 배송지 주소를 입력받는 기능이 필수였습니다. 대부분의 웹 서비스에서 주소의 일부분 만을 입력 받아 주소에 특화된 API를 활용하여 우편번호와 함께 입력되는 방식을 취하고 있고, 이 방식이 가장 적합한 것이라고 생각되었기에 이와 동일한 방식으로 구현하였습니다. 카카오 API를 통해 받아온 주소 데이터는 크지 않아 어플리케이션 내부에 저장되는 SQLite를 써 데이터베이스에 저장하였습니다. 그런 다음, 사용자가 배송지를 선택하면 해당 배송지에 체크 표시가 나타나며, 여러 배송지가 있는 경우 유용하게 사용됩니다. 이 선택된 배송지 정보는 Provider를 활용하여 개인 정보의 주소 칸으로 자동으로 전달됩니다. 이를 통해 사용자는 선택한 배송지를 간편하게 개인 정보에 적용할 수 있습니다. 이 프로세스를 원활하게 구현함으로써, 사용자 경험을 향상시키고 주소 관리를 효율적으로 할 수 있습니다.
